The IBM HandShaker (CONNECT) has at first been designed as a helping tool for the embedded system software and hardware developers, but eventually it has become what it is now - a fully-fledged universal DOS shell.

It is a Borland-style integrated environment application:

  • Multi-window text editor capable of processing files up to 1M.
    • vertical block support;
    • line draw mode;
    • expanding/shrinking tables;
    • math operations;
    • text formatting;
    • regular expression search (forward, backward, in several windows);
    • DOS/UNIX text mode support;
    • infinite Undo/Redo;
    • syntax highlighting;
    • user-defined external help file support;
    • customizable character conversion tables (XLAT) for different kinds of text.

  • Multi-window HEX editor.
    • insert/overwrite mode;
    • selected block operations;
    • search and replace;
    • block fill;
    • logical operations AND, OR, XOR.

  • Text viewer and HEX viewer.
    • view files of any size;
    • regular expression search (forward and backward);
    • copy to Clipboard;
    • overwrite edit mode in HEX viewer;
    • DOS/UNIX text mode support;
    • XLAT support.

  • dBASE file viewer.
    • regular expression search and replace (forward and backward);
    • field edit mode;
    • insert/delete records;
    • view memo fields;
    • data import/export;
    • copy/paste data to/from Clipboard;
    • XLAT support.

  • Built-in script language compiler. CONNECT built-in script language is a powerful tool to write serial transfer protocols and text processing programs with the ability to include binary code.
  • Built-in script language debugger.
    • trace into, step over commands;
    • debugging animation;
    • breakpoints;
    • watches window;
    • modify window;
    • "Turbo Debugger" - like CPU window for binary code debugging.

  • File Manager which looks much like the Norton Commander but is at least twice as powerful.
    • copy/rename/delete directories;
    • customizable archive processing;
    • Windows 95 long filename support;
    • display/copy/rename/delete file descriptions;
    • optimal fill copy mode and file splitting for removable disks;
    • file panel resizing;
    • Drag-And-Drop files and directories;
    • optional tree info auto-update after disk-changing operations;
    • save/restore panel selection on directory change;
    • select/unselect files from Clipboard;
    • search files by mask, size, time, attributes, contents;
    • approximate condition search, search specified drives or directories only, search in archives;
    • copy/rename/delete found files;
    • sort and highlight files/directories by mask;
    • advanced panel display and copy filters;
    • directory branch unfold;
    • history lists for all inputs;
    • customizable association lists for executable programs and external/internal viewers (Point-And-Shoot);

  • External application executor which is able to run various programming tools such as cross-assemblers, compilers etc. with the user-defined parameters depending on currently active window, or currently focused file name. For example, one can run an assembly code compiler for the newly edited text, view a report generated by the compiler in a special Message window, and quickly go to the source line which caused an error, if necessary. The executor has a lot of flexible features enabling to easily configure itself to run any type of compiler and other batch utilities.
  • Keyboard macro editor and player. Enables to quickly record, edit and play any sequence of keystrokes and commands and reassign any key or any sequence of keys in the editor, File Manager, terminal windows.
  • Clipboard window. One can copy/paste data from/to any of the program windows and input lines, including external program screens (Snipper).
  • ANSI.SYS-compatible terminal.
    • Back Scroll window;
    • built-in file transfer protocols
      (X-modem, Y-modem, Z-modem, Kermit, ASCII, BPlus);
    • the ability to specify almost any emulated terminal type by means of termcap-compatible files;
    • copy/paste data to/from Clipboard;
    • phone dialer queue.

  • Phone book with the dialer queue and the tree-like structure of records.
  • Miscallaneous tools:
    • programmer's calculator;
    • memory map viewer;
    • memory code viewer;
    • memory dump editor;
    • environment editor;
    • alarm clock;
    • ASCII table;
    • calendar;
    • print spooler with customizable external fonts;
    • CD-Player.

  • Context-sensitive help system.
    • regular expression search;
    • user-made help files;
    • inter-file cross-references;
    • external application execution.

  • HTML file viewer.
    • local and inter-file href links support;
    • regular expression search (forward and backward);
    • HTML tables display;
    • copy to Clipboard;
    • XLAT support.

  • Customizable program setup (colors, work directories, keyboard, mouse, country settings, confirmations, log file etc.).
  • Local menu (pop-up) called from almost every program window by pressing Alt+F10 keys, or right mouse button click.
Running CONNECT.

At DOS prompt type:

cn.exe [[/bxxxx]/s] [filename [filename]] [/ccfgname],

where

/s

optional

enables Snipper loading at the start

/bxxxx

optional

sets Snipper buffer size in bytes

filename

optional

specifies file(s) to load in a new Edit window

/ccfgname

optional

specifies an initial configuration file name

An initial configuration file name can also be specified by the environment variable CNCFG.
