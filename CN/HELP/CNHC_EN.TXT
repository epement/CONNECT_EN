[Instructions for the CN Help Compiler]

This program is designed to convert a text file into a help file that is
format-compatible with the file `CONNECT.HLP (Turbo Vision Help File)`.
The source file format for the `cnhc` utility is very simple. A topic
identifier must begin with the word `.topic` at the first position of the
line.  It may be followed by one or more identifiers used to reference this
specific context. An identifier consists of a sequence of characters from
the sets A..Z, a..z, _, and the digits 0..9, and must begin with a letter.
Identifiers are separated by commas.  For example:
┌────────────────────────────────┐
│.topic movs, movsb, movsw, movsd│
└────────────────────────────────┘
defines a single context for multiple references simultaneously.

A reference consists either of the link text enclosed in curly braces, or
of arbitrary text followed by a colon and the name of the topic identifier.
For example, the reference `{MOVS}` points to the topic declared by the
identifier `movs` (the case of the identifier is ignored).  Alternatively,
a reference such as {String transfers:`movs`} may be used, which also
points to the `movs` context.  If the help text itself needs to contain the
characters `{` and `}`, they must be replaced in the `cnhc` source file
with the characters corresponding to codes 243 and 242 (0xF3, 0xF2),
respectively.  If the link text itself needs to contain the character `:`,
it must be replaced with the character corresponding to code 247 (0xF7).
If a line of text needs to begin with a `.`, it must be replaced with the
character corresponding to code 249 (¨).  When the help file is displayed,
lines starting at the first character position are automatically wrapped to
fit within the window.  Lines starting with a space are displayed without
wrapping.  Lines with a `;` in the first position are treated as comments
and are ignored by the help file compiler.  To include a line in the help
file that actually begins with a `;`, it must be written as `;;` in the
source text.

It is possible to construct help text from multiple files. This is
accomplished using the keyword `.Include filename`, which must appear at
the very beginning of a line. Included files may be nested up to 50 levels
deep. The `filename` can be specified either as a full path or simply as a
filename. In the latter case, the included file is first sought in the
current directory, and then in the directory where the main help file is
located.

Special pseudo-topics `Color_1`, `Color_2`, through `Color_5` are reserved;
a link to one of these causes the link text itself to be displayed in the
corresponding color, as configured in the "Colors | Help window" dialog.

The pseudo-topic `Color_X` renders the link text invisible.

The pseudo-topic `AbsColor_NNN` sets the text attributes for the link,
where `NNN` is a decimal or hexadecimal value.

The pseudo-topic `OtherHelpFile` allows you to create a link to the default
(zero) context of a specified help file. For example:
┌────────────────────────────────────────┐
│Turbo Pascal help {TURBO:OtherHelpFile}.│
└────────────────────────────────────────┘
links to the file `TURBO.HLP`, provided it resides in the same directory as
the file containing the link. A different?and more flexible?method for
achieving this is described below.

If a link label begins with the character 0x19 (ASCII #25), then upon
activating that link, the system searches through the links and
pseudo-links within the target topic to find a match for the link text
itself. For example, suppose the following links are defined:

┌───────────────────────────────────────────┐
│{Text1:Text} {Text2:Text}  {Text3:Text}  │
└───────────────────────────────────────────┘
and a topic:
┌────────────────────────────┐
│.topic Text                 │
│                            │
│ {Text1:Color_1}            │
│Text1 Text1 Text1 Text1     │
│                            │
│ {Text2:Color_X}            │
│Text2 Text2 Text2 Text2     │
└────────────────────────────┘

When following the `Text1` link, the help system will display the topic
"Text" starting from the position of "Text1"; when following the `Text2`
link, it will display it starting from the position of "Text2",
respectively.  When following the `Text3` link, the help system will
display the topic "Text" from the very beginning.  When navigating to the
pseudo-link `Color_X`, the help system will place the cursor on the line
immediately following that pseudo-link; when navigating to visible links or
pseudo-links, the cursor is positioned directly on that link.

The total number of topics must not exceed 16,384.

It is possible to launch external programs directly from within the help
system.  To do this, you must create a special context (topic) where the
first line contains the keyword `$Execute`, the second line specifies the
program's path, and the third line contains the command line arguments for
the program being launched. All standard command-line macros are supported.
Thus, the following help snippet executes the command `arj -?` and opens a
viewer to display the resulting help output.

┌───────────────────────────────────────────────────────┐
│ {Arj} Help for ARJ Archiver Commands                  │
│                                                       │
│.topic arj                                             │
│ $Execute                                              │
│ arj $Msg $NoMsgWin $NoReread $NoReDraw; $View($Report)│
│ -?                                                    │
└───────────────────────────────────────────────────────┘

The command-line macro `$Help()` can accept a parameter specified directly
on the command line. When such a macro is invoked, the help system searches
for this parameter within the first context (topic) and navigates to it
if found. Thus, it is now possible to create links from one help file to a
specific context within another. 
┌───────────────────────────────┐
│ {The Built-in Assembler:basm} │
│                               │
│.topic basm                    │
│ $Execute                      │
│ $Help($cn\help\turbo.hlp)     │
│ $Word                         │
└───────────────────────────────┘

When a help file created in this manner is integrated into `CONNECT`,
pressing `F1` while within it opens the topic defined as the first entry in
the help file; pressing `Ctrl-F1` opens the second topic, and within that
topic, an `Incremental Search` is performed, searching through the links
for the word currently under the editor's cursor. Consequently, the first
topic (`topic`) in the help file should contain general links to the help
system's topics, while the second should contain a list of links
specifically for context-sensitive searching. The topic number can be
explicitly specified by adding the line `.topic First=0`.

The file `CNHCX.EXE` along with the auxiliary files `RTM.EXE` and
`DPMI16BI.OVL` enables the compilation of large help files on `286+`
machines equipped with extended memory.

[NOTE: Input files must have DOS line endings, CRLF (carriage return,
linefeed). The help compilers (CNHC or CNHCX) will fail with errror code
#6 if the input file has Unix line endings, LF (linefeed only). This is
especially important if saving to a git repository, which by default
saves files with Unix line endings.]
