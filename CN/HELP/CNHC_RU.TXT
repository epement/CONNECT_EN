  Эта  программа предназначена для перевода текстового файла в help-файл,
  совместимый  по  формату с файлом CONNECT.HLP (Turbo Vision Help File).
  Формат  исходного  файла  для утилиты cnhc очень простой. Идентификатор
  темы  должен  начинаться со слова .topic в первой позиции строки. После
  него  может  идти один или несколько идентификаторов для ссылки на этот
  контекст.  Идентификатор состоит из множества символов A..Z, a..z, _, и
  цифр  0..9  и  начинается с буквы. Идентификаторы разделяются запятыми.
  Hапример:
   ┌────────────────────────────────┐
   │.topic movs, movsb, movsw, movsd│
   └────────────────────────────────┘
  задает контекст сразу для нескольких ссылок.

  Ссылка  состоит  из текста ссылки, заключенного в фигурные скобки, либо
  из  произвольного  текста и отделенного двоеточием имени идентификатора
  темы. Hапример ссылка {MOVS} указывает на тему, объявленную идентифика-
  тором  movs  (регистр  идентификатора  игнорируется).  Также может быть
  ссылка  {Строчные  пересылки:movs}, которая также ссылается на контекст
  movs.  Если в тексте хелпа должны быть символы '{' и '}', то в исходном
  для утилиты cnhc файле их надо заменить соответственно на символы с ко-
  дами  243 и 242 (ґ, Ґ). Если в тексте ссылки должен быть символ ':', то
  его  надо  заменить  на 247 (і). Если строка текста должна начинаться с
  '.',  то  ее  надо заменить на символ 249 (ї). При показе хелпа строки,
  начинающиеся  с  первой  позиции выравниваются таким образом, чтобы они
  влезли  в окно. Строки, начинающиеся с пробела выводятся без выравнива-
  ния.  Строки, имеющие в первой позиции символ ';' являются комментарием
  и  не  обрабатываются компилятором help-файла. Для того, чтобы строка в
  help начиналась с ';', ее надо заменить в тексте хелпа на ';;'.

  Возможно составлять текст помощи из нескольких файлов. Для этого служит
  ключевое  слово .Include filename, которое должно стоять в первой пози-
  ции  строки. Возможна 50-кратная вложенность включаемых файлов. Filena-
  me  можно  задавать как в виде полного пути, так и в виде имени. В пос-
  леднем  случае включаемый файл ищется в текущем каталоге, а потом в том
  же каталоге, где был основной help-файл.

  Зарезервированы  специальные  псевдо-темы  Color_1, Color_2 .. Color_5,
  ссылка  на которые приводит к показу текста этой ссылки соответствующим
  цветом, устанавливаемом в диалоге Colors│Help window.

  Псевдо-тема Color_X делает тект этой ссылки невидимым.

  Псевдо-тема  AbsColor_NNN задает атрибут текста ссылки, где NNN - деся-
  тичное или Hex число.

  Псевдо-тема  OtherHelpFile позволяет сделать ссылку на нулевой контекст
  указанного файла помощи. Hапример:
   ┌────────────────────────────────────────┐
   │Turbo Pascal help {TURBO:OtherHelpFile}.│
   └────────────────────────────────────────┘
  ссылается  на  файл TURBO.HLP, лежащий в том же каталоге, что и ссылаю-
  щийся файл. Hиже приведен и другой, более гибкий способ это сделать.

  Если  имя ссылки начинается с символа  (#25), то при переходе на нее в
  ссылках и псевдо-ссылках темы ищется текст ссылки. Hапример пусть зада-
  ны ссылки:
   ┌───────────────────────────────────────────┐
   │{Text1:Text} {Text2:Text}  {Text3:Text}  │
   └───────────────────────────────────────────┘
  и тема:
   ┌────────────────────────────┐
   │.topic Text                 │
   │                            │
   │ {Text1:Color_1}            │
   │Text1 Text1 Text1 Text1     │
   │                            │
   │ {Text2:Color_X}            │
   │Text2 Text2 Text2 Text2     │
   └────────────────────────────┘
  При  переходе по ссылке Text1 хелп-система покажет тему  Text с позиции
  Text1,  а при переходе по ссылке Text2, соответственно с позиции Text2.
  При  переходе  по ссылке Text3 хелп-система покажет тему Text с начала.
  При  переходе  на псевдо-ссылку Color_X хелп-система поставит курсор на
  следующую  после  этой  псевдо-ссылки  строку,  при переходе на видимые
  ссылки и псевдо ссылки курсор становится на эту ссылку.

  Общее число тем не должно превышать 16384.

  Существует возможность из help-системы запускать внешние программы. Для
  этого надо создать специальный контекст (топик), первой строкой которо-
  го  будет  ключевое  слово  $Execute, второй - путь, а третей командная
  строка  запускаемой  программы.  Поддерживаются  все  макросы командной
  строки. Таким образом следующий фрагмент помощи запускает arj -? и отк-
  рывает вьювер для просмотра получившейся справки.
   ┌───────────────────────────────────────────────────────┐
   │ {Arj} Помощь по командам архиватора ARJ               │
   │                                                       │
   │.topic arj                                             │
   │ $Execute                                              │
   │ arj $Msg $NoMsgWin $NoReread $NoReDraw; $View($Report)│
   │ -?                                                    │
   └───────────────────────────────────────────────────────┘

  У  макроса $Help() коммандной строки может быть задаваемый в  командной
  строке параметр. При вызове такого макроса help-система ищет этот пара-
  метр в первом контексте (топике) и переходит на него, если находит. Та-
  ким образом теперь можно делать ссылки из одного help-файла на конкрет-
  ный контекст другого.
   ┌───────────────────────────────┐
   │ {The Built-in Assembler:basm} │
   │                               │
   │.topic basm                    │
   │ $Execute                      │
   │ $Help($cn\help\turbo.hlp)     │
   │ $Word                         │
   └───────────────────────────────┘

  При  подключении  сделанного  таким образом help-файла к CONNECT'у, при
  нажатии  в  нем F1 открывается тема, описанная в help-файле первой, при
  нажатии  Ctrl-F1  открывается вторая тема, и в ней делается Incremental
  search  слова  под  курсором редактора по ссылкам. Таким образом первая
  тема  (topic) help-файла должна содержать общие ссылки на темы хелпа, а
  вторая  -  список  ссылок для контекстного поиска. Hомер темы можно за-
  дать принудительно, написав .topic First=0.

  Файл CNHCX.EXE и вспомогательные файлы RTM.EXE и DPMI16BI.OVL позволяют
  на 286+ машинах с расширенной памятью компилировать большие help-файлы.
